# Audit Rules

Auditor must check:

- Builder followed the current phase.
- Builder only built allowed items.
- Builder avoided future-phase features.
- Builder used mock data only.
- Builder created clean components.
- Dashboard shows the required cards.
- Each card displays meaningful data.
- The app is understandable for a non-coder builder.
- There are no obvious UI problems.
- Builder produced a build report.

Audit result must be one of:

- Pass.
- Fix Needed.
- Blocked.
- Scope Creep.
- Needs User Decision.

A task is not Done until audit passes.
